# WeWeb 템플릿 제작 및 벤치마킹 기획안

## 1. 벤치마킹 개요 (imweb.me/theme)
imweb.me의 테마 라이브러리를 벤치마킹하여, WeWeb 프로젝트의 **Library 탭** 내에 사용자가 즉각적으로 적용할 수 있는 고퀄리티 UI 템플릿 컬렉션을 구축합니다. 
초기 진입 시 사용자의 시선을 사로잡는(WOW effect) 압도적인 디자인, 부드러운 마이크로 애니메이션, 세련된 타이포그래피가 핵심입니다.

## 2. 작업 사전 준비 및 준수 준칙
1. **기술 스택**: HTML, CSS(Vanilla), TypeScript
2. **사전 작업 (스킬 임포트 방침)**:
   - `D:\document\project\antigravity-awesome-skills\` 에서 웹 디자인 및 UI 구축에 필요한 스킬(예: `frontend-design`, `ui-skills`, `theme-factory` 등) 복사.
   - 복사된 스킬 디렉토리는 WeWeb의 `.gitignore` 또는 최상위 `.gitignore`에 등록하여 GitHub 업로드에서 제외.
3. **Monorepo 아키텍처 활용 (필수 준수)**:
   - 모든 UI 컴포넌트, 유틸리티 함수 등의 기능은 `D:\document\project\EverSite\Monorepo` (예: `@packages/ui`, `@packages/utils` 등) 라이브러리에서 `import`하여 사용합니다. 개별 프로젝트 내 독립적인 중복 컴포넌트 생성을 지양합니다.
4. **와이드 데스크톱(PC) 레이아웃 메인 & 반응형 스타일링 최우선**:
   - 순수 레이아웃 구축 및 미려한 스타일 디자인에 집중합니다.
   - **데스크톱 최적화 및 모바일 호환(Desktop-First Base)**: 모바일 형태를 억지로 늘린 UI가 아닌, **PC 해상도(1024px~1920px 이상) 기준의 와이드 뷰, 다단 그리드(Multi-column), 대형 시각 보정 레이아웃을 가장 기본 골격으로 작업**합니다. 이후 화면이 줄어듦에 따라 자연스럽게(Graceful Degradation) 모바일 UI로 스택킹(Stacking)되는 올바른 반응형 디자인을 반드시 적용합니다.
5. **추가 기능 제안 분리**:
   - API 연동, 복잡한 상태 관리(State Management) 등 추가 기능 구현이 필요한 경우 별도의 `.md` 문서로 정리해 유저의 판단 하에 진행 여부를 결정합니다.
6. **웹 에이전시 템플릿 설정 파일 도입**:
   - **`theme-config.json` 생성**: 각 템플릿의 변동 요소(Primary Color, 폰트 테마 등)를 별도의 JSON 파일로 생성하여 관리합니다. 
   - (참고: 프론트엔드와 완벽히 연동되는 Zero-Code 자동 렌더링 아키텍처 구성은 현재 단계에서 우선 보류하며, 향후 확장을 위해 설정 파일만 일차적으로 분리 보관합니다.)
7. **독립된 템플릿 폴더 구조 셋팅**:
   - 모든 템플릿 소스 코드는 **`D:\document\project\EverSite\WeWeb\template`** 디렉토리 내에 개별 폴더(예: `template/minimalist/`)로 분리하여 구성합니다. 코어 앱(Library 탭 등)에서는 이 `template` 폴더에 생성된 모듈과 설정값을 `import`하여 연결해 화면에 띄우는 구조를 강제합니다.
8. **AI 이미지 직접 생성 및 규격 최적화 (Placeholder 금지)**:
   - 빈 박스나 단순 회색 Placeholders 사용을 지양합니다. 자체 이미지 생성 툴(`generate_image` 등)을 사용하여 템플릿 디자인 컨셉과 톤앤매너에 어울리는 고품질 이미지를 직접 에셋으로 생성하여 렌더링합니다.
   - 각 이미지가 적용될 위치의 비율(Ratio)과 물리적 규격(Size)에 정확히 맞게 생성되도록 프롬프트를 조정합니다.

## 3. 템플릿 벤치마킹 카테고리 구성 (초기 3종)
템플릿은 목적에 맞게 형태가 뚜렷해야 하며, 다음 3가지 컨셉으로 시작합니다.

### A. Minimalist Studio (포트폴리오 및 에이전시)
- **주요 레이아웃**: 대규모 여백(White Space), 거대한 타이포그래피, 모노톤 + 포인트 컬러.
- **핵심 컴포넌트**: 
  - 풀스크린 비디오/이미지 Hero 섹션.
  - 비대칭(Asymmetric) 또는 Masonry 형태의 갤러리 그리드.

### B. Modern SaaS (IT 서비스 및 스타트업)
- **주요 레이아웃**: 신뢰감을 주는 구조형 레이아웃, B2B 특화 둥근 모서리(라운드 쉐이프) 카드.
- **핵심 컴포넌트**:
  - 글로우(Glow) 효과가 들어간 CTA(Call To Action) 프라이싱 카드.
  - 마우스 호버 시 입체감이 살아나는 Bento-Box 스타일의 피쳐(Feature) 리스트.

### C. Elegant E-Commerce (쇼핑몰 및 룩북)
- **주요 레이아웃**: 시각적 몰입감을 극대화한 이미지 중심 배열, 정돈된 상품 정보.
- **핵심 컴포넌트**:
  - 화면을 양분하는 Split-screen 구성 (절반 텍스트 / 절반 이미지 자동 슬라이드).
  - Hover Action이 결합된 상품 진열 그리드 및 퀵뷰 오버레이.
